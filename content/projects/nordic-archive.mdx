## Editorial System

The archive is built around a simple belief: a personal platform should make ideas easier to revisit. It should not behave like a feed. It should give images room to breathe, let essays accumulate slowly, and make the underlying structure visible without turning the site into a diagram.

The design language is dark, spare, and deliberate. Large type carries the voice. Thin rules carry the grid. The content files carry the future.

## Replaceable Content

Projects and posts are stored as MDX entries with metadata at the top. That makes the site easy to expand: add a file, update the import map, and the homepage, index pages, and SEO surfaces can all read from the same source.
