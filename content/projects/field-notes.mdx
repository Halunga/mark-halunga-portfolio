## Brief

Writing should not require a CMS for every project. This system uses MDX so posts and notes can live next to the codebase while still feeling designed, searchable, and connected to the rest of the site.

## Approach

Each content file exports metadata for the index page and then uses regular Markdown for the body. That keeps editing simple while allowing the front end to generate consistent lists, titles, dates, summaries, and SEO details.

## Outcome

The result is a content workflow that fits portfolios, essays, changelogs, documentation, research notes, and project journals without adding a heavy admin interface.
